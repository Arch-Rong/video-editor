# SDK / 第三方库对比：自研是否划算

> 配套可读 Canvas：`video-editor-sdk-comparison`（IDE 侧栏打开）。许可以各仓库实时 LICENSE 为准，上线前需法务复核。

## 1. 结论（先看这个）

| 路径 | 划算吗（对 Lumina-Muse 场景） |
| --- | --- |
| 全自研第二个剪映 | **不划算** |
| 整包嵌入第三方剪辑 SDK | **一般不划算**（前期快、后期锁） |
| **自研壳 + Schema + Embed，难模块用零件库/服务** | **划算（推荐）** |
| 把现有 Muse 剪辑板抽成 core 再补齐 | **最划算的起点** |

「自己写」应理解为：**控制工程模型与产品壳**，不是从零实现编解码与全部时间轴手势。

---

## 2. 方案地图

```
用户可拖拽的 NLE（剪映类）
├── 完整编辑器 SDK …… Twick / VideoFlow Editor / Elah / DesignCombo / OpenVideo / Shotstack Studio
├── 仅时间轴 UI ……… @xzdarcy/react-timeline-editor
└── 自研演进 ………… Muse EditPanel → video-editor

程序化 / 模板成片（不是自由时间轴产品）
└── Remotion

只负责「导出成片」
├── Shotstack / Creatomate API
└── 自建 FFmpeg 服务（推荐主后端）

底层零件
├── ffmpeg.wasm（小文件/演示）
└── WebCodecs / 各类 mediabunny 封装（中后期预览）
```

---

## 3. 逐个对比

评分维度（1–5，越高越好，主观评估）：

- **嵌 Muse**：塞进工作台、对齐故事板/资产是否顺
- **可控**：工程 JSON、导出链路、UI 定制是否在你手里
- **速度**：拿到「能演示的编辑器」有多快
- **成熟**：生产可供养程度

### 3.1 完整编辑器 SDK

#### Twick（`@twick/*`）

| 项 | 内容 |
| --- | --- |
| 是什么 | React 模块化剪辑 SDK：时间轴、画布、播放器、导出、AI 字幕 |
| 许可 | Sustainable Use License：嵌自有产品通常可；再卖 SDK / 竞品托管需商业授权 |
| 嵌 Muse | 3 |
| 可控 | 2 |
| 速度 | 5 |
| 成熟 | 3 |
| 优点 | 功能面全，上线演示快，包可拆 |
| 缺点 | 模型与导出路径易绑定；深定制≈fork；许可要确认 |
| 用不用 | **可做 ≤1 周 Spike**，不建议直接定为主路径 |
| 比自研 | 前两周更省；和故事板深度融合后不一定更省 |

#### VideoFlow

| 项 | 内容 |
| --- | --- |
| 是什么 | 分层：Apache core/renderer + 付费门槛的 React 整页编辑器 |
| 许可 | Core Apache-2.0；`react-video-editor` 小团队免费，公司常需付费 |
| 嵌 Muse | 3 |
| 可控 | 3（可只用 core） |
| 速度 | 4 |
| 成熟 | 3 |
| 优点 | 特效/转场/关键帧宣称强；可「买引擎不买整页」 |
| 缺点 | 整页 Editor 商务限制；仍要做资产桥接 |
| 用不用 | **中期可评估只引 renderer/core** |
| 比自研 | 若缺 WebGL 合成，买/引 core 可能比自研 compositor 划算 |

#### Elah

| 项 | 内容 |
| --- | --- |
| 是什么 | 框架无关引擎（时间轴、WebGL2、导出）+ React 薄绑定 |
| 许可 | ECL：嵌非「竞品剪辑 SaaS」可；托管剪辑平台要商用授权 |
| 嵌 Muse | 3 |
| 可控 | 4 |
| 速度 | 3 |
| 成熟 | 2（偏新） |
| 优点 | 架构思路正确（engine ≠ UI） |
| 缺点 | 生态/文档/稳定性未知 |
| 用不用 | **中期 PoC 候选**，不宜现在赌主路径 |
| 比自研 | 理念接近「推荐混合」；成熟度还不够替你们赌

#### DesignCombo `react-video-editor`

| 项 | 内容 |
| --- | --- |
| 是什么 | 开源「剪映/Canva 克隆」，Remotion 路线 |
| 嵌 Muse | 2 |
| 可控 | 2 |
| 速度 | 4 |
| 成熟 | 3（演示强，SDK 边界弱） |
| 用不用 | **当参考实现**，不建议整仓进生产 |
| 比自研 | 看起来省，维护与 Remotion 心智成本高 |

| 项 | 内容 |
| --- | --- |
| 是什么 | WebCodecs + Pixi 客户端渲染的 starter |
| 优点 | 本地导出可省服务器费 |
| 缺点 | 大文件/兼容性自担；双技术栈 |
| 用不用 | 验证「纯浏览器导出」时 Spike |

#### Shotstack Studio SDK（商用白标）

| 项 | 内容 |
| --- | --- |
| 是什么 | 可嵌入编辑器 + 云渲染一家包 |
| 速度 | 5 |
| 可控 | 2 |
| 费用 | 持续订阅/按量 |
| 用不用 | **默认不进主路径**（除非战略上接受外挂剪辑） |
| 比自研 | 产品冷启动最省；深度 AIGC 工作台不划算 |

#### `@xzdarcy/react-timeline-editor`

| 项 | 内容 |
| --- | --- |
| 是什么 | **只做时间轴**的 React 组件 |
| 许可 | 常见 MIT |
| 嵌 Muse | 4 |
| 可控 | 5 |
| 速度 | 4 |

### 3.3 程序化成片（不是剪映替代品）

#### Remotion

| 项 | 内容 |
| --- | --- |
| 是什么 | 用 React 写视频、模板、批量渲染 |
| 许可 | 公司使用需 Company License |
| 结论 | **推荐零件**（模板/批量），**禁止当 NLE 底座** |
| 与剪映关系 | 互补：用户自由剪 vs 程序填模板 |

---

### 3.4 云渲染 API（只导出）

#### Shotstack API / Creatomate

| 项 | 内容 |
| --- | --- |
| 是什么 | JSON 时间线 → 云端出片 |
| 结论 | **仅导出可选**；编辑器 UI 仍建议自有 |
| Creatomate | 更偏模板/营销自动化 |
| Shotstack | 更偏开发者 API + 可带 Studio |
| 比自建 FFmpeg | 量小省运维；量大或要强定制则自建更划算 |

---

### 3.5 底层零件

| 库 | 用途 | 建议 |
| --- | --- | --- |
| **服务端 FFmpeg** | trim/concat/烧字幕/滤镜 | **必配零件**，延续现有 merge 服务 |
| **ffmpeg.wasm** | 浏览器内导出 | 仅小文件/演示 |
| **WebCodecs 等** | 跟手预览、本地帧处理 | MVP 后性能瓶颈再上 |

---

### 3.6 自研演进（现有 Muse 剪辑板）

| 项 | 内容 |
| --- | --- |
| 已有 | 软切、单视频轨重排、胶片条、播放联动、合并导出 |
| 缺口 | 真多轨、字幕预览合成、转场滤镜、复杂导出一致性 |
| 结论 | **主路径** |
| 划算点 | 沉没成本 + 业务对齐；缺的能力用「零件」补，而不是推倒换 SDK |

## 4. 「自己写」成本怎么算才对

假设团队 1–2 名前端：

| 范围 | 粗量级 | 建议 |
| --- | --- | --- |
| 只维持单轨+导出（现状升级） | 数周 | 自研抽包 |
| 多轨+字幕+可用导出 | 1.5–3 人月 | 自研壳 + xzdarcy/FFmpeg |
| 接近剪映日常（特效关键帧滤镜） | 数人月+ | 评估 VideoFlow core / Elah / 自研 compositor |
| 真·剪映全集 | 年计 | 别做 |

整包 SDK：

| 阶段 | 成本特征 |
| --- | --- |
| 第 1–2 周 | 明显更省 |
| 嵌故事板、统一资产、定制交互 | 迅速赶超自研混合路径 |
| 许可证 / 云渲染账单 / fork 维护 | 持续隐形成本 |

## 5. 推荐决策树

```
要不要用户自由时间轴剪辑？
 ├─ 否 → Remotion 或 Creatomate/Shotstack 模板 API
 └─ 是 → 是否必须深度嵌 Muse 故事板？
      ├─ 是 → 自研壳 + editor-core（从剪辑板抽）
      │         ├─ 时间轴交互不足 → 试 xzdarcy
      │         ├─ 导出不够 → FFmpeg；尖峰可接云 API
      │         └─ 预览合成不够 → Spike VideoFlow core / Elah / WebCodecs
      └─ 否（独立剪辑 SaaS）→ 可严肃评估 Twick / Shotstack Studio
```

## 6. 行动建议（立刻可做）

1. **定主路径**：自有 Project JSON + 从 Muse 抽 core（不换整包 SDK）
2. **零件清单默认开**：服务端 FFmpeg、（可选）xzdarcy
3. **Spike 预算 1 周**：Twick **或** VideoFlow core，只回答「多轨预览能否省 1 人月」，回答完删 spike 分支
4. **并行可选**：模板批量成片用 Remotion / 云 API，与 NLE 解耦

---

## 7. 一句话

- **完整剪辑体验**：不要赌「某一个 SDK 等于剪映」。  
- **对你们最划算**：自己写「壳与模型」，别人写「零件」；现有剪辑板是资产，不是负担。
