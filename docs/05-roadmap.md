# 里程碑规划

目标：先交付「能真正剪完并导出」的 Web 剪辑器，再逐步逼近剪映体验。

---

## Phase 0 — 地基（1–2 周）

**目标**：仓库可跑、模型定稿、从现有剪辑板抽出核心算法。

- [ ] 初始化 monorepo（或单体）与 lint/test 基线
- [ ] 固化 `Project` schema（zod）与 `schemaVersion=1`
- [ ] 迁移：`chat-main TimelineSnapshot → Project`
- [ ] `editor-core`：duration、split、trim、reorder、normalize、history
- [ ] 空白 Editor Shell：顶栏 + 三栏布局 + 空时间轴
- [ ] 文档评审通过（本目录）

**完成标准**：无 UI 特效也能用单测证明核心剪辑命令正确。

---

## Phase 1 — MVP（约 4–6 周）对标「能用的剪映精简版」

**目标**：用户可完成 导入 → 单/双轨剪辑 → 字幕 → 导出。

### Must

- [ ] 素材上传与素材库
- [ ] 1 视频轨 + 1 音频轨 + 1 文字轨
- [ ] 片段：添加、移动、裁剪手柄、分割、删除
- [ ] 胶片条（可先做简化版）
- [ ] 预览播放与播放头联动
- [ ] 基础字幕（文案、字号、颜色、位置）
- [ ] Undo / Redo + 本地自动保存（IndexedDB）
- [ ] 导出 mp4（可先复用现有 merge/trim 服务，限制「无复杂滤镜」）
- [ ] Embed API 草案落地，Lumina-Muse 可替换入口做联调

### Should

- [ ] 画布 transform（位移/缩放）
- [ ] 时间轴缩放与边缘自动滚动
- [ ] 导出进度与失败提示

**完成标准**：内部试用能剪一条 30–60s 竖屏成片并下载。

---

## Phase 2 — v1（约 6–8 周）对标「接近日常剪映」

- [ ] 多视频轨叠画
- [ ] 转场（fade 等）
- [ ] 滤镜 / 基础调色
- [ ] 音频波形、淡入淡出、音量包络简化版
- [ ] 磁吸对齐、涟漪删除选项
- [ ] 关键帧（opacity / transform）
- [ ] 代理预览与性能优化
- [ ] 工程云同步（可选，接宿主账号）
- [ ] 服务端渲染队列（复杂工程走 FFmpeg 图谱）

**完成标准**：无需桌面剪映即可完成常见短视频修饰；嵌入宿主稳定。

---

## Phase 3 — v2（持续）差异化与智能

- [ ] ASR 自动字幕
- [ ] 模板系统（可替换槽位素材）
- [ ] AI 辅助：智能粗剪 / 卡点 / 抠像
- [ ] 素材商城接口预留
- [ ] WebCodecs / WebGL 高级预览
- [ ] 协作与版本历史
- [ ] 移动端只读预览或轻编辑

---

## 建议迭代节奏

| 周期 | 产出 |
| --- | --- |
| 每周 | 可演示增量 + 文档同步 |
| 每 Phase 末 | 体验走查 + 导出回归 + 性能抽检 |
| 每次 schema 变更 | 写 migrate + 单测 |

## 风险与缓解

| 风险 | 缓解 |
| --- | --- |
| 浏览器解码性能不够 | 代理媒体、可见窗抽帧、导出上云 |
| 跨域无法抽帧 | 统一媒体代理 / CORS；降级占位图 |
| 与 Muse 剪辑板双轨维护 | 尽快切 Embed，旧面板只留兼容层 |
| 导出效果 ≠ 预览 | 尽早统一 Render Plan；关键特性同路径验证 |
| 范围膨胀向剪映全集 | 严格按 P0/P1；模板与 AI 放到 Phase 3 |

## 从现有代码搬迁顺序（推荐）

1. `timelineUtils` 纯函数 → `editor-core`
2. `types` → `editor-schema`（并做字段改名）
3. `timelinePlaybackStore` 思路 → playback clock
4. `useClipReorder` / filmstrip 队列 → `editor-ui` 热路径模块
5. 导出 `mergeVideosOrThrow` → `editor-export` 适配器
6. `EditPanel` UI 退役，改挂 Embed
